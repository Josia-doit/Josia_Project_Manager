# Josia 项目管理器 使用说明

## 📌 软件简介

Josia 项目文件管理器，专为使用AI生成代码的编程小白开发，可快速搭建项目结构。

通过 AI 生成标准 JSON，一键自动创建子目录、文件并写入代码，告别手动新建、复制粘贴的繁琐。

## 🌟 核心功能

- 📁 项目管理：打开/关闭项目、重命名项目文件夹

- 📝 文件编辑：内置代码编辑器，支持语法高亮、保存、撤销、重做

- 🧩 批量写入：通过 AI 生成的 JSON 指令，一键生成完整项目结构（子目录+文件+代码）

- 🗂️ 目录管理：新建文件、新建文件夹、删除选中项、重命名文件/文件夹

- 📤 批量导出：将当前项目所有代码文件批量导出为 JSON 格式，便于备份、共享及二次编辑

- 📦 一键打包 EXE：支持单文件/多文件两种打包模式，打包后自动清理临时文件，无需手动操作

- 📖 内置帮助：包含完整使用说明与 AI 指令模板，一键复制即可使用

- 🎨 现代界面：扁平化风格简洁 UI，操作流畅，适配编程小白使用

- 🧪 运行测试：快速验证项目运行状态，可自行扩展测试逻辑

## 🖱️ 基本操作流程

### 1. 📁 设定项目目录

点击工具栏「设定目录」按钮，选择一个文件夹作为项目根目录，后续所有操作均基于该目录进行。

### 2. ✏️ 项目重命名

右键点击顶部项目标题（显示为「📁 项目名称」），即可对项目文件夹进行重命名操作。

### 3. 📤 生成项目结构

点击工具栏「帮助」按钮，切换至「AI 指令模板」页面，复制模板内容，发送给 AI 生成符合格式要求的 JSON 指令（包含文件路径和代码内容）。

### 4. 🚀 批量写入

点击工具栏「批量写入」按钮，将 AI 生成的 JSON 指令粘贴至弹出窗口，点击「执行写入」，程序将自动创建所有子目录、文件，并写入对应代码，无需手动操作。

### 5. 📝 编辑文件

在左侧文件树中，单击任意文件即可在右侧编辑器中打开，支持语法高亮、文本编辑、保存、撤销、重做等操作，无需切换外部编辑软件。

### 6. 📤 批量导出

点击工具栏「批量导出」按钮，程序将自动将当前项目所有代码文件（不含图标等非代码文件）导出为 JSON 格式，可用于备份、共享或二次编辑。

### 7. 📦 一键打包 EXE

点击工具栏「打包 EXE」按钮，按提示完成以下操作即可打包：

- 选择输出目录：指定生成的 EXE 文件存放位置

- 输入 EXE 名称：自定义可执行文件的名称

- 选择打包模式：单文件（适合单独部署）、多文件（适合调试）

- 点击「开始打包」，等待打包完成，程序会自动清理 build、dist 临时目录及 .spec 配置文件，仅保留输出目录中的 EXE 文件。

### 8. 🧪 运行测试

点击工具栏「运行测试」按钮，可快速验证当前项目的运行状态，该功能可根据自身需求扩展测试逻辑。

## 🔧 常见问题解决

- **启动报错**：若出现代码编辑器相关报错，大概率是 QsciScintilla 颜色参数错误，当前版本已修复，替换最新版 code_editor.py 即可正常启动。

- **打包后冗余文件**：无需手动清理，程序会在打包完成后，自动删除 build、dist 临时目录及 .spec 配置文件，仅保留你指定输出目录中的 EXE 文件。

- **文件操作失败**：优化了文件读写异常处理，若出现创建/删除/重命名失败，会弹出错误提示，可根据提示检查文件是否被占用。

- **批量导出无内容**：批量导出仅支持代码文件（如 .py、.js、.html 等），图标、图像等非代码文件不会被导出。

- **JSON 写入失败**：请确保粘贴的 JSON 格式正确，可参考帮助文档右侧的 AI 指令模板，避免格式错误导致写入失败。

## 💡 温馨提示

- 使用批量写入功能前，请确保已设定项目目录，避免文件生成路径错误。

- 打包 EXE 时，建议选择空的输出目录，避免与其他文件混淆。

- 若需备份项目，可使用批量导出功能，将项目导出为 JSON 文件，后续可通过批量写入功能快速恢复。

- AI 生成 JSON 时，请严格按照帮助文档右侧的模板格式，确保「path」和「content」字段完整，避免语法错误。
